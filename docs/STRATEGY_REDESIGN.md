## 2. 전략 1: Momentum v2 (개선)

### 2.1 현재 문제점 (백테스트에서 확인)

1. **거래 빈도 극저**: 55거래일에 1~20건 → 하루 0~0.4건
2. **리테스트 미구현**: PRD에 명시된 "돌파→리테스트→재돌파" 미적용, 첫 돌파에 진입하여 위꼬리 피해
3. **누적 거래량 200%**: 장 초반에는 불가능, 오후에만 신호 → 시간대 편향
4. **고정 손절 -0.8%**: 돌파 직후 일시 되돌림에 노이즈 손절

### 2.2 개선 설계

```
[Momentum v2 — 전일 고점 돌파 + 리테스트 진입]

■ 진입 조건 (3단계 상태 머신)
  STATE 1 "대기": 현재가 > 전일 고점 → STATE 2 전환, 돌파가 기록
  STATE 2 "리테스트 대기": 현재가가 전일 고점 ±0.3% 이내로 되돌림
                          최대 대기 시간: 돌파 후 30분 (초과 시 STATE 1 리셋)
  STATE 3 "재돌파 확인": 리테스트 후 다시 돌파가 상회 + 양봉 확인 → 매수 신호

■ 거래량 필터 (시간대 보정)
  현재: 누적 거래량 >= 전일 총거래량 × 2.0
  변경: 동시간대 누적 거래량 >= 전일 동시간대 × 1.5
  → 오전에도 거래량 조건 충족 가능

■ 손절 (동적)
  현재: 고정 -0.8%
  변경: max(리테스트 저점 - 0.3%, 진입가 × 0.992)
  → 리테스트 저점이 지지선 역할, 최소 -0.8% 보장

■ 익절
  유지: TP1 +3% → 50% 매도 → 손절 본전 이동 → trailing -1%

■ VWAP 방향 필터 (VWAP 전략의 유산)
  현재가 > VWAP일 때만 매수 허용
  → VWAP이 독립 전략으로는 실패했지만, 방향 필터로는 유효
```

### 2.3 기대 효과

- 거래 빈도: 리테스트 대기 30분 + 동시간대 거래량 → 기존 대비 1.5~2배 증가 기대
- 승률: 위꼬리 피해 감소 → 기존 45~62%에서 55~65% 기대
- 손절 품질: 동적 손절이 노이즈 필터링 → PF 개선

### 2.4 config 변경

```yaml
strategy:
  momentum:
    volume_ratio: 1.5           # 2.0→1.5 (동시간대 기준으로 변경)
    stop_loss_pct: -0.008       # 유지 (동적 손절의 최소값)
    retest_band_pct: 0.003      # 신규: 전일 고점 ±0.3% 리테스트 밴드
    retest_timeout_minutes: 30  # 신규: 리테스트 최대 대기
    vwap_filter: true           # 신규: VWAP 방향 필터 ON
```

## 3. 전략 2: Pullback v2 (개선)

### 3.1 현재 문제점

1. **4개 조건 동시 충족이 너무 엄격**: 1분봉 기준으로 사실상 발생 불가에 가까움
2. **종목 의존도**: 고변동 종목(바이오 PF 1.30, 조선 PF 1.29)에서만 작동
3. **5MA ±0.5% 터치**: 코스닥 종목의 1분봉 변동폭 대비 밴드가 너무 좁음
4. **+4% 조건**: 대형주에서는 장중 +4% 자체가 드묾

### 3.2 개선 설계

```
[Pullback v2 — 강세 종목 조정 후 반등]

■ 조건 완화
  조건 1: 당일 시가 대비 +4% → +2.5% (코스피), +3% (코스닥)
          → config에서 시장별 분리 또는 단일 +2.5%로 하향
  조건 2: 5MA ±0.5% → 10MA ±1.0%
          → 이동평균 주기 늘려 안정화, 터치 밴드 2배 확대
  조건 3: 음봉→양봉 전환 → 유지 (핵심 패턴)
  조건 4: MA20 정배열 → MA10 정배열 (20분→10분으로 단축)
          → 장 초반에도 작동 가능

■ 타임프레임 전환
  현재: 1분봉 기준 모든 조건
  변경: 5분봉 기준으로 조건 판단
  → 1분봉 노이즈 제거, 패턴 신뢰도 상승
  → CandleBuilder에서 5분봉은 이미 생성 중

■ 종목 적합성 필터
  ATR(14)/종가 >= 2.5% 인 종목에서만 Pullback 활성화
  → 스크리닝 단계에서 이미 계산되는 값 활용
  → 저변동 대형주(삼성전자 등)에서 자동 비활성화

■ 손절 (기존 유지 + 동적 옵션)
  고정: -1.8% (기존 최적화 결과)
  동적: min(10MA - 0.5%, 진입가 × 0.982)

■ 익절
  유지: TP1 +3% → 50% 매도 → trailing -1%
```

### 3.3 기대 효과

- 거래 빈도: 조건 완화 + 5분봉 전환 → 기존 대비 2~3배 증가 기대
- 종목 필터: 고변동 종목 자동 선별 → 손실 종목(반도체/삼성전자) 자동 제외
- 승률: 5분봉으로 노이즈 감소 → 기존 47~67%에서 55~65% 기대

### 3.4 config 변경

```yaml
strategy:
  pullback:
    min_gain_pct: 0.025         # 0.04→0.025 (조건 완화)
    stop_loss_pct: -0.018       # 유지
    ma_short: 10                # 신규: 단기 이평 (기존 5)
    ma_long: 10                 # 신규: 정배열 판정 이평 (기존 20)
    ma_touch_band: 0.01         # 0.005→0.01 (터치 밴드 확대)
    timeframe: "5m"             # 신규: 5분봉 기준
    min_atr_pct: 0.025          # 신규: 종목 적합성 필터
```

## 4. 전략 3: FlowStrategy (신규 — 수급추종)

### 4.1 설계 근거

한국 시장의 구조적 특성:
- 기관/외국인의 순매수 방향이 장중 주가를 지배
- 개인 투자자 대비 정보 비대칭이 큰 시장
- **체결강도(매수체결량/총체결량)** 120% 이상은 강한 매수 수급 시그널
- 키움 WS로 실시간 체결 데이터가 이미 유입 중 → 인프라 추가 비용 없음

### 4.2 전략 설계

```
[FlowStrategy — 실시간 수급 쏠림 포착]

■ 진입 조건 (AND)
  1. 체결강도 >= 120% (직전 5분 기준)
     체결강도 = (매수 체결량 / 총 체결량) × 100
     → 키움 WS 틱의 매수/매도 구분 활용
  2. 5분 거래량 >= 20분 평균 × 2.5배
     → RealtimeScanner의 로직 확장 (기존 3배→2.5배)
  3. 현재가 > VWAP (방향 필터)
  4. 현재가 > 당일 시가 (장 시작 이후 상승 중)

■ 진입 타이밍
  조건 1+2 동시 충족 시점의 다음 양봉 확인 후 진입
  → 급등 꼭대기가 아닌 수급 확인 후 진입

■ 손절
  진입가 × (1 - 1.5%) = -1.5%
  또는 VWAP 이탈 시 즉시 손절 (VWAP이 지지선 역할)
  → min(진입가 × 0.985, VWAP - 0.2%)

■ 익절
  TP1 +3% → 50% 매도 → trailing -1.5%
  (수급 추종은 추세가 이어질 수 있으므로 trailing을 넓게)

■ 시간 제한
  09:30~14:30 (장 초반 30분은 수급 신호 불안정, 종료 40분 전 진입 금지)

■ 데이터 소스
  - 틱 데이터: kiwoom_ws.py에서 이미 수신 중
  - 체결강도 계산: tick의 매수/매도 구분 필드 활용
    → 키움 WS FID 체계에서 체결 구분(매수/매도) 확인 필요
  - 5분 거래량: CandleBuilder의 5분봉 volume 활용
```

### 4.3 구현 범위

```
신규 파일:
  strategy/flow_strategy.py      — FlowStrategy(BaseStrategy)
  data/flow_tracker.py           — 체결강도/거래량 급증 실시간 추적

수정 파일:
  core/kiwoom_ws.py              — 체결 구분(매수/매도) 파싱 추가
  screener/strategy_selector.py  — FlowStrategy 선택 로직 추가
  config/settings.py             — FlowConfig 파라미터 추가
  config.yaml                    — flow 전략 파라미터 섹션
  main.py                        — flow_tracker 파이프라인 연결
```

### 4.4 config 추가

```yaml
strategy:
  flow:
    min_strength_pct: 120.0     # 체결강도 하한 (%)
    volume_surge_ratio: 2.5     # 5분 거래량 / 20분 평균
    stop_loss_pct: -0.015       # 손절 -1.5%
    trailing_stop_pct: 0.015    # 트레일링 -1.5% (넓게)
    signal_start: "09:30"       # 신호 활성 시작
    signal_end: "14:30"         # 신호 활성 종료
    vwap_filter: true           # VWAP 방향 필터
```

### 4.5 백테스트 제약

**중요**: 체결강도는 실시간 틱 데이터의 매수/매도 구분이 필요한데,
`intraday_candles` DB에는 OHLCV만 저장돼 있어 **기존 데이터로는 백테스트 불가**.

대안:
1. **페이퍼 트레이딩으로 직접 검증** (2~4주)
2. 체결강도를 거래량 급증(volume surge)으로 대체한 근사 백테스트
3. 향후 틱 레벨 데이터 수집 후 정식 백테스트

→ **Phase 1에서는 거래량 급증 기반 근사 버전으로 구현하고,
   페이퍼 트레이딩으로 체결강도 데이터를 축적한 후 Phase 2에서 정식 적용.**

## 5. 폐기 전략 처리

### 5.1 VWAP — 방향 필터로 전환

독립 전략 폐기. `strategy/vwap_strategy.py`는 삭제하지 않고 유지하되,
StrategySelector에서 선택 대상에서 제외.

**VWAP 방향 필터 함수를 BaseStrategy 또는 유틸에 추가:**
```python
def is_above_vwap(candles: pd.DataFrame) -> bool:
    """현재가가 VWAP 위인지 판정. Momentum/Flow의 보조 필터."""
    if candles is None or candles.empty or "vwap" not in candles.columns:
        return True  # VWAP 데이터 없으면 통과 (안전 폴백)
    vwap = candles.iloc[-1]["vwap"]
    close = candles.iloc[-1]["close"]
    return close > vwap if vwap and vwap > 0 else True
```

### 5.2 ORB — 완전 폐기

`strategy/orb_strategy.py`는 삭제하지 않고 `strategy/_deprecated/`로 이동.
config.yaml에서 ORB 관련 파라미터는 주석 처리.

## 6. StrategySelector 재설계

### 6.1 현재 (순차 우선순위)
```
ORB(갭) > Momentum(섹터ETF) > VWAP(횡보) > Pullback(폴백)
```

### 6.2 변경 (병렬 조건 + 독립 활성화)

```
전략 선택 로직 변경: 상호 배타적 선택 → 독립 활성화

하루에 1개 전략만 선택하는 현재 구조를 유지하되,
선택 기준을 시장 상황에서 "종목 특성"으로 전환:

1. 스크리닝에서 상위 1종목 선정 (기존)
2. 해당 종목의 당일 특성을 분석:
   - 전일 고점 돌파 가능성 높음 → Momentum
   - 당일 갭 상승 + 고변동 → Pullback
   - 거래량 급증 감지 → Flow
3. 기본값: Momentum (가장 검증된 전략)
4. 폴백: 조건 없으면 "당일 매매 없음"
```

### 6.3 장중 전략 전환 (Phase 2 검토)

현재는 하루 1전략 고정이지만, 향후:
- 08:30 스크리닝 → Momentum으로 시작
- 장중 수급 급변 감지 시 → Flow로 전환
- 이건 복잡도가 높으므로 Phase 2에서 검토

## 9. 리스크 관리

1. **과적합 경계**: 기존 데이터에 맞추면 미래에 안 먹힘.
   → 16종목 중 8종목으로 최적화, 나머지 8종목으로 검증 (walk-forward)
2. **전략 간 상관관계**: Momentum과 Pullback이 같은 날 같은 종목에서 동시 신호 가능
   → 하루 1전략 고정 구조로 충돌 방지
3. **Flow 전략 미검증**: 백테스트 불가이므로 페이퍼 2주 필수
   → 소액(100만원) 실전 전 반드시 페이퍼 데이터 확보
