# NXT(야간/시간외) 데이터 활용 가능성 조사

> 작성: 2026-05-13 | 목적: 키움 OpenAPI+에서 NXT 데이터 수신 여부 확인

---

## 1. KRX NXT 개요

**KRX NXT (New eXtended Trading)**는 2024년 7월 22일 도입된 KRX 야간 거래 시스템이다.

| 세션 | 시간 | 내용 |
|------|------|------|
| 프리마켓 (Pre-market) | 08:00 ~ 08:30 | 정규장 전 단일가 매매 |
| 정규장 | 09:00 ~ 15:30 | 연속 매매 |
| 시간외 단일가 | 15:40 ~ 16:00 | 정규장 종가 기준 단일가 |
| NXT 야간 | 16:00 ~ 20:00 | 야간 연속 매매 (일부 종목) |

---

## 2. 키움 OpenAPI+ REST (ka10001 등) — NXT 지원 여부

### 2-1. ka10001 (주식 현재가 조회)

**지원 여부: 부분 지원 (정규장 전용으로 추정)**

- ka10001은 정규장(09:00~15:30) 체결가 기준으로 응답한다.
- NXT 세션(16:00~20:00) 중에도 API 엔드포인트는 응답하나 **반환 가격이 정규장 종가 또는 마지막 체결가**일 가능성이 높다.
- NXT 실시간 체결가를 가져오는지 여부는 **실 API 호출로 확인이 필요**하다.

**필요 조치**: NXT 시간대(16:30)에 ka10001 호출 → 응답의 `stck_prpr` 필드가 정규장 종가와 동일하면 NXT 미지원.

### 2-2. NXT 전용 REST API

키움 OpenAPI+ 공개 문서(2024~2026 기준) 기반으로 확인된 사항:

| API | 용도 | NXT 지원 |
|-----|------|---------|
| `ka10001` | 주식 현재가 | 정규장 전용 (추정) |
| `ka10002` | 주식 일별 OHLCV | NXT 미포함 (일봉 기준) |
| `ka10119` | 주식 시간대별 체결 | 정규장 09:00~15:30만 조회 |
| `ka20001` | ETF 현재가 | 정규장 전용 |

**NXT 전용 API 미확인**: 키움 OpenAPI+ 공개 문서에서 NXT 전용 REST 엔드포인트는 현재까지 확인되지 않는다. 키움 HTS(Hero 영웅문)에서는 NXT 데이터를 제공하지만 OpenAPI+에의 노출 여부는 별도 확인이 필요하다.

---

## 3. 키움 WS 0B — NXT 시간 체결 수신 여부

### 3-1. 구독 지속 여부

WS 연결은 session 기반이므로 NXT 시간에도 연결이 유지되면 데이터를 수신할 수 있다. 그러나 **현재 시스템은 `is_ws_active_hours()` 판단으로 장외 시간에 재연결을 생략**한다(`core/kiwoom_ws.py` `_listen_loop`). NXT 시간(16:00~20:00)이 `is_ws_active_hours()` 범위에 포함되지 않으면 WS가 자동 재연결되지 않는다.

**필요 조치**: `utils/market_calendar.py`의 `is_ws_active_hours()` 기준 시간 확인. NXT 체결 수신이 필요하면 해당 함수 범위 확장 필요.

### 3-2. 0B 메시지 실제 수신 여부

NXT 체결이 WS 0B 타입으로 수신되는지 여부는 **키움 서버가 NXT 체결도 0B로 push하는지**에 달려 있다. 공개 문서에 명시되지 않으며, raw 로그 수집으로 확인이 필요하다.

**추정**: KRX NXT는 KRX 거래 시스템과 별도 세그먼트로 운영되므로 키움 WS가 NXT 체결을 0B로 push하지 않을 가능성이 있다.

### 3-3. WS NXT 전용 실시간 타입

키움 OpenAPI+ 공개 문서에서 확인된 실시간 타입:
- `0B`: 체결 (정규장)
- `0D`: 호가 (정규장)
- `00`: 주문체결 통보

**NXT 전용 WS 타입 미확인**: NXT 전용 실시간 타입(예: `0E`, `0N` 등)은 공개 문서에 기재되지 않음.

## 4. 활용 방안 (지원 시)

NXT 데이터 접근이 확인될 경우의 활용 시나리오:

### 4-1. 프리마켓(08:00~08:30) 가격/거래량

- 프리마켓 체결가 ↑ = 당일 모멘텀 강화 신호
- 스크리닝 점수 가산 (+score): 프리마켓 거래량 × (프리마켓 종가 / 전일 종가 - 1)
- 구현 위치: `screener/pre_market.py` 에서 08:00 REST 조회 추가

### 4-2. 시간외 종가(15:40~16:00)

- 정규장 종가 대비 등락 → 다음날 갭 방향 예측
- 전일 고가 돌파 기준가(prev_high) 보정에 활용 가능
- 신뢰도 제한: 거래량 낮아 대표성 부족

### 4-3. NXT 야간(16:00~20:00)

- 야간 체결가 흐름 → 다음날 갭업/갭다운 예측
- 활용 난이도: 높음 (거래량 극소, 조작 리스크, API 미지원 가능성)
- 현재 시스템 아키텍처(당일 09:05~12:00 진입)와 연계점 제한적

## 6. 결론 및 권고사항

### 단기 (현재 시스템 범위 내)

1. **NXT 데이터는 현재 시스템에 통합하지 않는다.** 공개 API 문서 미확인 항목이 많고, 검증 비용 대비 전략 기여도가 불확실하다.
2. **is_ws_active_hours() 점검**: NXT 시간이 포함되지 않아 WS 연결이 끊기면 05:35 분봉 수집 전 재연결이 필요한 케이스가 있을 수 있다.

### 중기 (검증 후)

1. 키움 고객센터에 OpenAPI+에서 NXT 체결 수신 가능 여부 문의.
2. 확인되면 `is_ws_active_hours()` 범위를 20:00까지 확장 + 프리마켓 스크리닝 스코어 추가.
3. NXT 거래량 데이터를 전일 거래량 대비 필터(volume_ratio=2.0) 계산에 포함할지 별도 그리드 검증.

---

> **검증 미완료 항목**: 이 문서는 공개 문서 + 구조 추론 기반이며, 실 API 호출 결과로 업데이트가 필요하다. 각 "추정"/"미확인" 항목은 페이퍼 트레이딩 기간 중 실 로그로 확인할 것.
