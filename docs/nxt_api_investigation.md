# NXT(야간/시간외) 데이터 활용 가능성 조사

> 작성: 2026-05-13 | 실측 업데이트: 2026-05-14 16:29 (NXT야간 세션)
> **결론: 키움 OpenAPI+로 NXT 데이터 취득 불가. 향후 다른 데이터 소스 검토 필요.**

---

## 1. KRX NXT 개요

**KRX NXT (New eXtended Trading)**는 2024년 7월 22일 도입된 KRX 야간 거래 시스템이다.

| 세션 | 시간 | 내용 |
|------|------|------|
| 프리마켓 (Pre-market) | 08:00 ~ 08:30 | 정규장 전 단일가 매매 |
| 정규장 | 09:00 ~ 15:30 | 연속 매매 |
| 시간외 단일가 | 15:40 ~ 16:00 | 정규장 종가 기준 단일가 |
| NXT 야간 | 16:00 ~ 20:00 | 야간 연속 매매 (일부 종목) |

## 2. 키움 OpenAPI+ REST — 실측 결과 (2026-05-14 16:29, NXT야간)

`scripts/test_nxt_api.py` 실행 결과. 저장 파일: `logs/nxt_test_20260514_162944.json`

### 2-1. TR ID 후보 탐색 결과

| TR ID | 설명 | HTTP | raw_keys | 판정 |
|-------|------|------|---------|------|
| `ka10001` | 주식현재가 (정규장) | 200 | `stk_cd`, `stk_nm`, `cap`, `flo_stk`... | **현재가(`cur_pric`) 필드 없음** — NXT 시간 미갱신 |
| `ka10003` | 시간외단일가 후보 | 200 | `cntr_infr`, `return_code`, `return_msg` | 데이터 없음 (빈 응답) |
| `ka10004` | NXT 현재가 후보 | 200 | `return_code`, `return_msg` only | 데이터 없음 |
| `ka10023` | 시간외종가 후보 | 200 | `return_code`, `return_msg` only | 데이터 없음 |
| `ka10086` | 시간외차트 후보 | 200 | `return_code`, `return_msg` only | 데이터 없음 |

**핵심**: `ka10003~ka10086` 모두 HTTP 200을 반환하지만 `return_code`/`return_msg`만 포함한 빈 응답 — NXT 데이터를 제공하지 않는다.

### 2-2. ka10001 가격 필드 (NXT 시간 16:29 호출)

```
high_pric : +299500   (정규장 고가)
low_pric  : -282000   (정규장 저가)
base_pric :  284000   (전일 기준가)
trde_qty  : 39295913  (정규장 거래량)
upl_pric  : +369000   (상한가)
cur_pric  : (없음)    ← NXT 현재가 미제공
```

`cur_pric`(현재가) 필드가 응답에 없다. NXT 시간에 ka10001은 정규장 통계 필드만 반환하며 **NXT 실시간 체결가를 노출하지 않는다.**

## 3. 키움 WS — 실측 결과 (2026-05-14 16:29, NXT야간)

### 3-1. 연결 및 구독

- WS LOGIN: **성공**
- 0B(체결) + 0D(호가) 구독: **성공** (구독 명령 자체는 에러 없음)
- 0B 수신: **0건** — NXT 체결이 0B 타입으로 push되지 않음
- 0D 수신: **0건** — 호가 데이터도 없음

### 3-2. is_ws_active_hours() 확인

- NXT야간(16:00~20:00) → `is_ws_active_hours() = False`
- WS 재연결 로직이 NXT 시간대를 비활성으로 처리 — 장 종료 후 연결 끊기면 재연결 없음
- **현재 시스템 영향 없음**: 분봉 수집(15:35), 강제청산(15:10) 등 모든 운영 작업이 15:35 이전에 완료됨

## 4. 결론 (실측 확정)

**키움 OpenAPI+로 NXT 데이터 취득 불가 — 확정.**

| 항목 | 상태 (실측) | 근거 |
|------|------------|------|
| NXT REST 현재가 (`ka10001`) | **미제공** | NXT 시간 `cur_pric` 필드 없음 |
| NXT 전용 REST TR (`ka10003~ka10086`) | **미제공** | 빈 응답 (`return_code`/`return_msg` only) |
| WS 0B NXT 체결 수신 | **미수신** | 15초 대기 중 0건 |
| WS 0D NXT 호가 수신 | **미수신** | 15초 대기 중 0건 |
| WS NXT 전용 타입 | **없음** | 구독 응답에 NXT 타입 없음 |

## 5. 향후 데이터 소스 검토 방향

키움 OpenAPI+ 이외의 NXT 데이터 소스 후보:

| 소스 | 방법 | 난이도 | 비고 |
|------|------|--------|------|
| 네이버 금융 | HTML 크롤링 | 중 | ToS 검토 필요, 실시간성 제한 |
| 카카오 금융 | API 비공개 역공학 | 상 | 법적 리스크 |
| KRX 공식 | `http://data.krx.co.kr` | 중 | 지연 데이터(T+0), 무료 |
| 증권사 HTS 패킷 | 패킷 스니핑 | 상 | 키움 외 증권사 OpenAPI 검토 필요 |
| LS증권 OpenAPI | REST/WS | 중 | NXT 지원 여부 별도 확인 필요 |

**단기 우선순위**: 현재 전략(09:05~12:00 정규장 진입)에서 NXT 데이터의 실질 기여는 낮다. 페이퍼 트레이딩 실적 확인 후 필요 시 재검토.

## 6. 시스템 영향

- **현재 운영 코드 변경 없음**: NXT 데이터 미사용 확정이므로 `obi_filter_enabled: false` 및 WS 0D 관련 TODO 상수는 **정규장 0D 확인** 후 별도 처리.
- `is_ws_active_hours()` 확장 불필요: NXT 체결이 WS로 수신되지 않으므로 확장 이득 없음.
- `ka10003~ka10086` TR ID: 사용 계획 없음.
