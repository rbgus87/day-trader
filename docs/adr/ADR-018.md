# ADR-018: 상한가 즉시 청산

## 일자
2026-04-21

## 상태
ACCEPTED

## 결정

포지션의 **현재가 ≥ 상한가** 도달 시 **즉시 시장가 매도**한다. 체결 실패 시 **stop_loss를 상한가 × 0.99로 상향**하여 되돌림 손실을 제한한다.

```yaml
strategy:
  momentum:
    limit_up_exit_enabled: true
    limit_up_pct: 0.30                # 전일종가 × 1.30
    limit_up_stop_floor_pct: 0.99     # 체결 실패 시 stop = 상한가 × 0.99
```

상한가는 **전일 종가 × 1.30을 한국 호가 단위로 절사**하여 계산. 08:05 전일 OHLCV 갱신 시 유니버스 전 종목에 대해 일괄 산정한다.

기존 청산 경로 (stop_loss / trailing / breakeven / forced_close) **최상위 우선순위**.

## 배경

Breakeven Stop (ADR-017) 도입 후 baseline PF 4.28 유지 중, **상한가 도달 케이스**의 처리가 구조적으로 취약했다:

- 현행 Chandelier trailing (최고가 − ATR × 1.0) 은 상한가 근방에서 stop이 상한가 아래에 머물러, 상한가 이탈(매물 출회) 시점까지 기다리다 되돌림 손실 발생
- BE3는 +3% 도달 후 entry+1% 로 stop 상향하지만, +30% 급등 상황에서도 트리거는 변하지 않음
- 실무적으로 상한가 도달 종목은 **다음날 갭 오픈 리스크**가 크며, 당일 상한가에서 매도 체결하는 것이 기대 수익 확정 경로

- 나머지 3건 — 상한가 도달 후 되돌림이 컸다면 어차피 BE3/SL로 빠졌을 케이스 (흡수)
- 청산 분포 총합은 **248건**으로 일관성 유지

## 구현

### 1. 호가 유틸 (`core/price_utils.py` 신규)

2023-01-25 개편 한국 주식 호가 단위 (KOSPI/KOSDAQ 공통):

| 가격 | 호가 |
|---|---|
| < 2,000원 | 1원 |
| 2,000 ~ 5,000 | 5 |
| 5,000 ~ 20,000 | 10 |
| 20,000 ~ 50,000 | 50 |
| 50,000 ~ 200,000 | 100 |
| 200,000 ~ 500,000 | 500 |
| ≥ 500,000 | 1,000 |

```python
def calculate_limit_up_price(prev_close: float, limit_pct: float = 0.30) -> int:
    """전일 종가 × (1 + limit_pct)를 호가 단위로 절사한 상한가."""
    if prev_close <= 0:
        return 0
    return round_down_to_tick(prev_close * (1.0 + limit_pct))
```

### 2. 리스크 관리 (`risk/risk_manager.py`)

- `register_position` — `limit_up_price` 파라미터 수용, 포지션 dict에 저장
- `check_limit_up(ticker, price)` — `current_price >= limit_up_price` 판정
- `raise_stop_to_limit_up_floor(ticker)` — 체결 실패 후 stop을 상한가 × 0.99로 상향, `limit_up_exit_failed` 플래그로 재시도 차단

### 3. 운영 경로 (`gui/workers/engine_worker.py`)

- `_limit_up_map: dict[str, float]` — 08:05 전일 OHLCV 갱신 시 각 종목의 상한가 산정·저장
- `signal_consumer.register_position` 호출 시 `limit_up_price=self._limit_up_map.get(ticker)` 주입
- `tick_consumer` 루프에서 **stop_loss 체크 전에 상한가 우선 판정**:
  - 성공 — `execute_sell_stop(..., exit_reason="limit_up_exit")` → `settle_sell` → `trade_executed` emit
  - 실패 — `raise_stop_to_limit_up_floor` 호출, 이후 기존 SL/trailing 경로가 처리

### 4. 백테스트 (`backtest/backtester.py`)

- `_setup_strategy_day` 에서 `prev_day_df.iloc[-1]["close"]` 기반 당일 상한가 계산 → `self._current_limit_up`
- 진입 시 `position["limit_up_price"]` 기록
- 청산 루프 최상위에 `high >= limit_up_price` 분기 → 상한가에서 청산 (`exit_reason="limit_up_exit"`)
- 비용 모델은 기존 `apply_sell_costs` 재사용 (슬리피지 0.03% + 수수료 0.015% + 세금 0.15%)

### 5. GUI (`gui/widgets/dashboard_tab.py`)

- `REASON_CODES` 에 `"limit_up_exit": "상한"` / `"breakeven_stop": "BE"` 매핑 추가
- 체결 테이블 사유 컬럼에 "상한" 표시, 툴팁 원본 `limit_up_exit`

## 특성

- **확정 수익**: 상한가 도달 + 체결 성공 시 +30%대 수익 확정 (실측 +8.64% 거래당 평균은 진입가 기준)
- **되돌림 방어**: 체결 실패(유동성 부족) 시 stop을 상한가 × 0.99 로 올려 최소 +28.7% 이상 확보
- **갭 리스크 차단**: 당일 상한가 → 익일 장 초반 갭 하락 리스크 회피
- **기존 엣지 보완적**: BE3/trailing 과 독립 동작, 최상위 우선순위로 간섭 없음
- **발동 빈도 낮음**: 연 15건 / 248거래 (6.0%) — 자주 발동하는 메커니즘이 아닌 tail 이벤트 포착

## 리스크

- **체결 실패 후 재상승 케이스**: stop을 상한가 × 0.99로 올렸는데 잠깐 이탈 후 다시 상한가 복귀 → stop에 터치되어 +28.7%에서 청산 (상한가 +30% 대비 -1.3%p 손실). 시뮬상 발동률이 낮아 허용.
- **유동성 얕은 종목의 상한가**: 매도 체결 가능성이 낮음. ATR ≥ 6% 필터 + 거래대금 ≥ 50억 (ADR-012) 이 1차 방어선.

## 페이퍼 체크 항목

- 실제 상한가 도달 종목의 체결 성공률 (시뮬은 100% 전제)
- 체결 가격 슬리피지 (시뮬 0.03%, 실측과 갭)
- raise_stop_to_limit_up_floor 발동 빈도
