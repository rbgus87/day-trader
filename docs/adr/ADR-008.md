# ADR-008: 알림 정책 통일 + on/off 토글 도입

## 일자
2026-04-15

## 결정
모든 텔레그램 알림을 10종 이벤트로 분류하고, 각각을 config + GUI 양쪽에서 on/off 가능하게 한다. 체결 알림 포맷은 `notifier.send_execution`으로 통일 (paper/live 공통).

## 배경
- **silent gap 사건 (4/14)**: `engine_worker._notify_execution` + paper/order_manager 내부 알림이 동시 발신되어 텔레그램 중복. 커밋 `16da917`로 1차 수정했으나 정책 자체가 명시적이지 않아 재발 위험.
- **포맷 이원화 (Phase 3-A)**: `paper_order_manager`는 `notifier.send` 자유 텍스트, `order_manager`는 `notifier.send_execution` 구조화. 두 경로의 메시지 모양이 다름.
- **on/off 부재**: 운영자가 알림 피로감 있어도 조정 불가. config 파일에 토글 키가 없음.

## 구현 (커밋 6개)

### 1) NotificationConfig dataclass + config.yaml (`991448b`)
- `config/settings.py`: `NotificationConfig` (frozen dataclass, 10 필드 모두 default True)
- `AppConfig.from_yaml()` 로더에 `notifications` 섹션 파싱 추가
- `config.yaml`: notifications 섹션 신설 (10 키 명시)

### 2) 알림 호출부 전수 분기 + WS 재연결 성공 알림 (`e960ec8`)
- `engine_worker.py`: 6개 호출부에 `if config.notifications.X` 게이트
- `_safe_refresh_ohlcv`: 성공 알림 신설 + 실패 알림 게이트
- 각 OrderManager / WS 인스턴스화에 `notifications_config` 전달
- `core/kiwoom_ws.py`:
  - `_ws_notify_enabled()` 헬퍼
  - `_last_disconnect_at` 기록
  - 재연결 성공 시 `[복구] WS 자동 재연결 성공 (시도 N회, 끊김 HH:MM:SS)` 알림

### 3) 체결 알림 포맷 통일 (`d5c28ed`)
- `notifier.send_execution()`에 `mode='live'/'paper'`, `reason`, `pnl`, `pnl_pct` 옵션 추가
- `paper_order_manager`: 자유 텍스트 `send` → `send_execution(mode='paper')` 통일
- 테스트 갱신 (`mock_notifier.send_execution` 호출 검증)

### 4) GUI 알림 토글 탭 (이번 커밋)
- `gui/widgets/strategy_tab.py`: 「알림 정책」 그룹박스 신설
- 10개 QCheckBox + `_NOTIFICATION_FIELDS` 정의
- `load_config` / `get_config`에 notifications 직렬화 통합
- 단위 시뮬: 토글 변경 → `get_config` notifications dict 갱신 확인

### 5) ADR-008 + CLAUDE.md (이번 커밋)

## 영향
- 운영자 알림 피로감 → 개별 항목 끄기 가능
- 메시지 포맷 일관 → 텔레그램 검색·필터 용이
- WS 자동 복구 가시성 ↑ (이전엔 실패만 알렸음)
- baseline PF 2.91 영향 없음 (알림은 라이브 전용 부속)

## 단위 시뮬 결과
- `trade_execution` 토글: ON 1회 / OFF 0회 (`PaperOrderManager.execute_buy`)
- GUI 토글 변경 → `get_config()` notifications dict 즉시 반영
- `load_config({"notifications": {...}})` → 체크박스 상태 동기화
