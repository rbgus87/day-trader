# ADR-006: 일일 리셋 — 명문화 + 자동화 (시나리오 C)

## 일자
2026-04-15

## 결정
일일 리셋과 전일 OHLCV 갱신을 **이중 안전망**으로 구현:
- **1차 (정상)**: 운영자 매일 GUI 재시작 (`python gui.py`)
- **2차 (안전망)**: 자정 01분 자동 리셋 + 08:05 OHLCV 갱신
- **감지**: GUI 시작 시 24시간 이상 가동 감지 → 텔레그램 안내

## 배경
Phase 2-B 완료 시점 baseline 8종 대조에서 남은 2종:

- **#3 일일 리셋**: 15:10 `_force_close` 가 `active_strategies = {}` 로 비운 뒤 **재등록 로직이 없음**. 같은 프로세스를 다음날까지 돌리면 전략이 없어 매매 불가.
- **#4 전일 OHLCV**: startup 시 1회만 주입. 연속 운영 시 Day N의 "전일 고가"가 Day 0 기준으로 stale.

두 건 모두 "매일 GUI 재시작" 전제이지만, 이 전제가 문서화 안 됐고 운영자가 잊거나 밤샘 가동하면 silent fail.

## 대안
| 안 | 장점 | 단점 |
|---|---|---|
| A. 명문화만 | 단순, 구현 비용 0 | 운영자 의존, 잊으면 시스템 깨짐 |
| B. 자동화만 | 운영자 개입 없이 동작 | 자동 리셋 시점 상태 변화 리스크 |
| C. 둘 다 (선택) | 이중 안전망 | 구현 복잡도 약간 증가 |

## 결정 이유
- 운영자 직접 재시작이 **가장 깨끗한 상태** 보장 (WS 재연결, 메모리 정리)
- 자동화는 운영자 부재 시에만 동작하는 안전망
- 둘 다 있으면 어느 쪽 실패해도 안전
- "자동 리셋 때 포지션 보존" 원칙으로 이상 상황 최소화

## 구현 (커밋 `744290c` + `d95322c`)

### 1) 리팩터 — startup 로직 분리
`engine_worker.py` 에 재사용 helper 추가:
- `_load_universe()` — `universe.yaml` 로드 + `_ticker_markets` 갱신
- `_register_active_strategies(stocks)` — Momentum 전략 인스턴스 등록
- `_refresh_prev_day_ohlcv(stocks=None)` — 일봉 조회 + `set_prev_day_data` 주입

startup 인라인 코드를 이 helper로 치환. daily_reset / 08:05 cron이 같은 helper 재사용.

### 2) 자정 00:01 `_daily_reset`
1. `risk_manager.reset_daily_counters()` — `_daily_pnl=0`, `_halted=False` (포지션 보존)
2. `_daily_halt_notified = False`
3. `active_strategies` 비어있으면 재등록, 아니면 각 strategy `.reset()`
4. `_refresh_prev_day_ohlcv()` 호출
5. 텔레그램 안내 `[자동] 일일 리셋 완료 — 60종목, 카운터 초기화`

### 3) 08:05 `_refresh_prev_day_ohlcv`
- 토큰 갱신 (08:00) 직후 실행
- 실패 시 `[경고] 전일 OHLCV 갱신 실패 — ...` 긴급 알림, 기존 데이터로 운영 지속

### 4) `_check_uptime_sanity` (startup)
- `logs/.last_startup` 파일에 이전 시작 시각 저장
- 현재 시각과 비교하여 24시간 이상이면 텔레그램 안내
- 항상 현재 시각으로 파일 갱신

### 5) `risk_manager.reset_daily_counters()` 추가
- 기존 `reset_daily()` 는 `_positions.clear()` 포함 → 자정 리셋엔 부적합
- 포지션 보존한 채 카운터만 리셋하는 메서드 신설

## 포지션 보존 이유
- 정상 흐름에선 15:10 `force_close`가 이미 모든 포지션 청산
- 하지만 force_close 실패·네트워크 장애·프로세스 재시작 등으로 오버나이트 포지션 잔존 가능
- 자정 자동 리셋이 포지션까지 비우면 **복구 대상 실종** → `reset_daily_counters`로 안전하게 카운터만

## 영향 및 검증
- baseline 8종 일치율 **6/8 (75%) → 8/8 (100%)** 도달 예상
- pytest 190 passed / selftest 7/7
- 백테스트 PF 2.91 / 185건 / 청산 분포 동일 유지
- 단위 시뮬:
  - `_daily_reset` → 60종목 재등록 + `_trade_count=0` + `prev_day_high` 갱신 + notifier 1회 송신
  - `_check_uptime_sanity` (26시간 전 기록) → 안내 알림 1회
