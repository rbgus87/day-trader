# ADR-003: main.py 폐기 — GUI 단일 경로 확립

## 일자
2026-04-15

## 결정
`main.py` CLI 엔트리포인트를 완전히 폐기한다. 모든 라이브 운영은 `gui.py` + `gui/workers/engine_worker.py` 경로로 통일한다. `selftest`는 `gui.py --selftest` 또는 `python selftest.py`로 호출.

## 배경
Phase 2 단계 2 대조(`docs/spec/live_baseline_comparison.md`) 결과, baseline 8종 중 5종 불일치가 관측되었고 그 중 2건(15:10 강제청산 PnL=0 버그, 시장 필터 미구현)이 **`main.py`에만 존재**하는 문제였다.

- `main.py:384-401` `force_close` — `price=int(pos.get("entry_price", 0))` 로 항상 진입가 청산 → 모든 청산 PnL 0으로 기록되는 치명 버그
- `main.py` 전체에 `MarketFilter` 사용이 없어 KOSPI/KOSDAQ 약세에도 매수 진행

`engine_worker.py`는 이 두 항목이 모두 구현되어 있으나, 두 엔트리포인트가 **서로 다른 매매 동작**을 함. 코드 이원화가 baseline 불일치의 근본 원인.

## 대안
1. `main.py`와 `engine_worker.py`를 양쪽 다 수정해 동기 유지 → **지속적인 drift 위험**
2. `main.py`를 `engine_worker.py`에 맞춰 수정 → drift 위험 여전히 존재
3. `main.py` 폐기, GUI 단일 엔트리 (선택) → 단일 진실점 확립

## 결정 이유
- GUI 모드가 실운영 경로. CLI 모드는 현재 사용되지 않음.
- 이원화 구조 자체가 silent drift의 공급원.
- 단일 경로로 향후 sync 누락 구조적 차단.
- `selftest` 경로는 `gui.py --selftest` / `selftest.py` 직접 실행 둘 다 이미 지원됨 (별도 이관 불필요).

## 영향
- CLI로 "프로세스만" 띄우는 헤드리스 운영은 불가 (필요 시 후속 ADR로 헤드리스 모드 재도입)
- main.py 의존 테스트·스크립트 없음 확인 (`grep -rn "from main\|import main\b"` 0건)

## 상태
ACCEPTED — 커밋 `723555c` (main.py 삭제, 참조 정리)
