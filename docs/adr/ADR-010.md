# ADR-010: Pure Trailing + ATR 6% 유니버스 채택

## 일자
2026-04-16

## 상태
ACCEPTED

## 결정

1. **TP1 분할매도 시스템 폐기** — `atr_tp_enabled: false`
   - 기존 TP1(ATR×3.0, 실효 +12~25%)은 당일 단타에서 도달률 1.2%로 사실상 비활성
   - 분할매도 제거, 진입 즉시 전량 trailing 활성 (Pure trailing)

2. **ATR Chandelier trailing multiplier 1.0** — `atr_trail_multiplier: 2.5 → 1.0`
   - 시뮬 5종(1.0/1.5/2.0/2.5/3.0) 중 ×1.0이 PF 최고 (1.76 자본 가중)
   - trailing_stop 빈도 17.8% (×2.5의 7%에서 대폭 증가)
   - 약세장 PF 2.04 (×2.5의 1.88보다 우수)

3. **ATR ≥ 6% 유니버스 필터** — 60종목 → 41종목
   - 저변동성 대형주(삼성전자 5.0%, 현대차 5.4% 등) 제외
   - PF 2.68 → 3.21 (1주 가중), 약세장 PF 0.83 → 1.42

4. **Stop Loss 현 상태 유지** — ATR×1.5 (clamp 1.5~8%)
   - 51/60 종목이 상한 8%에 클램핑 → 사실상 -8% 고정
   - 추후 별도 ADR에서 상한 조정 검토

## 영향

### PF 변화
- 이전 baseline: PF 2.61 (1주 가중, 60종목, TP1+분할매도)
- 새 baseline: PF 3.21 (1주 가중, 41종목, Pure trailing)
- 자본 가중(100만원): PF ~1.70

### 청산 분포 변화
- 이전: forced_close 93.9% / stop_loss 4.9% / tp1_hit 1.2% / trailing_stop 0%
- 이후(예상): forced_close ~72% / stop_loss ~10% / trailing_stop ~18%

### 코드 변경
- `backtest/backtester.py`: `atr_tp_enabled=false` 시 진입 즉시 `tp1_hit=True` 설정 (Pure trailing)
- `risk/risk_manager.py`: `update_trailing_stop`에서 Pure trailing 시 `tp1_hit` 없이도 동작
- `gui/workers/engine_worker.py`: exit_reason 판별에 Pure trailing 분기 추가
- `config.yaml`: TP1 관련 제거, multiplier/ATR 필터 갱신
- `config/settings.py`: 기본값 동기화
- `config/universe.yaml`: 60 → 41종목

### Dead config 처리
- `tp1_pct`, `tp1_sell_ratio`, `trailing_stop_pct`: TradingConfig에 잔존 (코드 호환), config.yaml에서 제거
- `atr_tp_multiplier`, `atr_tp_min_pct`, `atr_tp_max_pct`: `atr_tp_enabled=false`로 미참조

## 보완 (2026-04-16): 청산 튜닝 완결

### 1. atr_trail_multiplier 미세 그리드
- x0.5/0.7/0.85/1.0/1.2 시뮬 → **x1.0 최적 확인**
- x0.5: 너무 타이트, 수익 거래 조기 청산 (PF 2.20)
- x1.0: 수확 체감 정점 (PF 3.27 wrapper)
- 결과: reports/multiplier_fine_grid.md

### 2. atr_trail_min_pct/max_pct 그리드
- R0(현재 2/10%), R1(1/5%), R2(1/3%), R4(2/15%) 시뮬
- 41종목 ATR 6%+이므로 max 축소 시 전종목 고정값 클램핑 → 종목별 차별화 소실
- **min2%/max10% 최적 확인** (ATR 기반 동적 6~10%가 고정값보다 우수)
- 결과: reports/trail_range_grid.md

### 3. stop_loss 고정 -8% 전환
- S0~S6 7종 시뮬 → ATR multiplier는 dead parameter (전종목 max 클램핑)
- `atr_stop_enabled: false`, `stop_loss_pct: -0.080`으로 단순화
- 실측 PF: 3.02 → **3.28** (+0.26, ATR→고정 전환으로 일부 종목 조기 손절 제거)
- 결과: reports/stop_loss_grid.md

### 시뮬 방식 결정 (갭 분석)
- 시뮬 wrapper와 실제 backtester 청산 우선순위 차이 발견
- 시뮬 PF는 실측 대비 ~0.2 과대 추정 (상대 순위 유효)
- 향후: wrapper로 빠른 탐색 → 실제 backtester로 최종 측정
- 결과: reports/sim_gap_analysis.md

### 최종 baseline (청산 튜닝 완결)
- **PF 3.28** (1주 가중, 실측)
- 거래수 279
- 청산: forced_close 208(74.6%) / stop_loss 65(23.3%) / trailing_stop 6(2.2%)
- 유니버스 41종목 (ATR >= 6%)

## 미래 검토 항목

1. **유니버스 갱신 주기** — ATR은 시간에 따라 변동, 분기 1회 재생성 필요
2. **자본 사이징** — 1주 가중 PF(3.28) vs 자본 가중 PF(~1.7) 괴리 해결
3. **진입 조건 재검증** — ADX/거래량/buy_time_end를 새 baseline 기준으로 재최적화
