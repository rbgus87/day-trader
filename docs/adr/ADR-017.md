# ADR-017: Breakeven Stop (BE3) 도입

## 일자
2026-04-17

## 상태
ACCEPTED

## 결정

포지션의 **peak_return ≥ 3%** 도달 시 stop_loss를
**진입가 × (1 + 1%) = entry × 1.01**로 상향한다 (1회, 비가역).

```yaml
strategy:
  momentum:
    breakeven_enabled: true
    breakeven_trigger_pct: 0.03   # +3% 도달 시 발동
    breakeven_offset_pct: 0.01    # stop을 entry+1%로
```

기존 Chandelier trailing (ADR-010, 최고가 − ATR × 1.0)과 **공존 `max()`**.

## 배경 — 페이퍼 1일차 관찰

2026-04-17 페이퍼 시작 1일차 체결 3종목 중:
- 코오롱티슈진 진입 후 -5.57%, 15:10 강제청산까지 자리 차지
- 미래에셋벤처투자 +20.54% 기회 → 자리 소진으로 진입 불가

이 중 "올랐다 되돌림" 타입의 손실을 방어하기 위한 구조적 개선.

## 구현 범위

### 운영 로직 (risk_manager.py:120-130, update_trailing_stop 확장)
```python
if getattr(self._config, "breakeven_enabled", False) and not pos.get("breakeven_active", False):
    entry = pos["entry_price"]
    peak = pos["highest_price"]
    trigger = getattr(self._config, "breakeven_trigger_pct", 0.03)
    if entry > 0 and (peak - entry) / entry >= trigger:
        offset = getattr(self._config, "breakeven_offset_pct", 0.01)
        be_stop = entry * (1.0 + offset)
        pos["stop_loss"] = max(pos["stop_loss"], be_stop)
        pos["breakeven_active"] = True
```

## 특성

- **올랐다 되돌림 방어**: peak 3% 경험 후 되돌리는 손실 거래를 +1% 수익으로 전환
- **자리 비움 부수 효과**: forced_close 71.8% → 59.8% (-12%p)
- **약세장 개선**: 약세 PF 2.16 → 3.11 (+44%)
- **강세장 개선**: 5.02 → 5.88 (+17%)
- **1회 작동, 비가역**: 한 번 발동 후 stop은 trailing이 더 올릴 수는 있어도 내려가지 않음
- **처음부터 하락 종목은 방어 못 함**: 이 케이스는 시뮬에서 NTS 보조 시도했으나 PF 악화로 보류

## NTS 보류 사유

- NTS 발동 건의 D0 매칭 반등 비율 3.6~11.5%로 "정확히" 자름
- 그러나 NTS 청산가가 장 후반 forced_close 청산가보다 평균적으로 나쁨
- 조합 시나리오 모두 BE3 단독 대비 PF -0.64~-0.74 악화
- 페이퍼 실데이터로 **자리 비움 → 추가 진입** 효과 실측 후 재검토
