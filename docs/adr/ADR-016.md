# ADR-016: 돌파 폭 하한 3% 도입

## 일자
2026-04-17

## 상태
ACCEPTED

## 결정

MomentumStrategy 진입 조건에 **최소 돌파폭 3%**를 추가한다.

```yaml
strategy:
  momentum:
    min_breakout_pct: 0.03
```

기존 조건 7·10(전일 고가 돌파)을 다음과 같이 강화:

```python
breakout_pct = (price - prev_day_high) / prev_day_high
if breakout_pct < self._config.min_breakout_pct:
    return None
```

## 배경

기존: `price > prev_day_high` — 1원만 넘어도 OK. 미세 돌파 노이즈가 진입 품질을 희석할 가능성.

## 근거 — B00 기준 돌파 폭 버킷별 PnL 분포

| 돌파 폭 | 거래수 | 거래당 PnL | 승률 |
|---|---|---|---|
| 0 ~ 3% | 52 | -24 | 51.9% (엣지 없음) |
| 3% 이상 | 224 | +1,295 | 57.1% (엣지 전부) |

→ 3% 미만 52건(전체 18.8%)이 평균 엣지 부재. 제거 시 거래수 12% 감소에도 총 PnL 증가.

## 시장 국면별 효과

| 시나리오 | 강세 PF | 횡보 PF | 약세 PF |
|---|---|---|---|
| B00 | 5.02 | 2.22 | 1.55 |
| B30 | 4.95 | 2.76 (+0.54) | **2.16 (+0.61)** |

강세장 거의 불변, **약세장 PF +39% 개선**. 횡보장도 +24%. 미세 돌파는 약세/횡보에서 가짜 돌파가 많음.

## 구현 범위

- `config.yaml`: `strategy.momentum.min_breakout_pct: 0.03`
- `config/settings.py`: `TradingConfig.min_breakout_pct: float = 0.03` + `from_yaml` 로딩
- `strategy/momentum_strategy.py`: 조건 7, 10에 `breakout_pct < min_bp` 가드
- `backtester.py`: 변경 없음 (MomentumStrategy가 config 참조)

## 특성

- **종목 무관 구조적 개선**: universe 변경 영향 없음
- **페이퍼 영향**: 과거 데이터 기반 진입 품질 강화 — 09:39 이후 첫 ADX/거래량 충족 분봉에서 돌파폭 3% 미만이면 진입 불가
- **side effect**: 일부 "돌파 직후 미세 상승" 후 본격 상승하는 종목 놓칠 수 있으나 시뮬상 순효과 +
