# ADR-005: 쿨다운·max_trades 훅 재검증 + force_close on_exit 보완

## 일자
2026-04-15

## 결정
`strategy.on_entry()` / `strategy.on_exit()` 훅은 이미 `engine_worker.py`에서 호출 중. `_force_close` 경로만 누락이라 해당 지점에 `on_exit()` 호출 추가.

## 배경
Phase 2 단계 2 대조에서 "라이브는 `on_entry/on_exit` 호출 지점이 없음"으로 기술 → **분석 오류**. 실제 코드:

- `engine_worker.py:735` `strategy.on_entry()` — execute_buy 성공 직후 호출
- `engine_worker.py:546` `strategy.on_exit()` — stop_loss/trailing_stop 청산 직후 호출
- `engine_worker.py:838-863` `_force_close` (15:10) — **호출 누락**

TP1 경로는 부분 매도로 포지션이 유지되므로 on_exit 호출 불필요 (백테스트 동일).

## 백테스터와의 비교
`backtest/backtester.py`에서 `strategy.on_exit()` 호출:
- L257 stop_loss
- L305 TP1+forced_close 잔여
- L375 trailing_stop
- L393 TP1 상태 forced_close 잔여
- L412 TP1 미히트 forced_close

라이브는 3/5 매칭(stop_loss, trailing_stop은 동일 경로, forced_close만 누락).

## 변경
`engine_worker.py:_force_close`에서 각 포지션 청산 직후 `strat_info["strategy"].on_exit()` 호출 추가. `active_strategies = {}` 로 인스턴스가 곧 버려지지만, 훅 계약 일관성을 위해 호출.

## 영향
- 런타임 동작 변화: 거의 없음 (force_close 직후 active_strategies={}로 인스턴스 폐기)
- 코드 계약 일관성: ✅
- baseline 재현: PF 2.91 / 185건 유지 확인 (`c089ce5` 커밋 검증)

## 상태
ACCEPTED — 커밋 `c089ce5`
