# ADR-001: day-trader 재조립 결정

## 일자
2026-04-15

## 결정
백테스트 환경을 유일한 baseline으로 삼고, 라이브 운영 부속을 처음부터 의도적으로 재조립한다.

## 배경
4/13~4/15 페이퍼 운영 중 11개 silent gap 발견:

- numba exe 누락
- DB 기록 버그
- time_stop 불일치
- tp2 dead code
- trailing 폴백 불일치
- 5개 archive 전략
- 16개 dead config
- 텔레그램 중복
- 포지션 사이징 미동기화
- CLAUDE.md 부정확
- 분할매수 백테스트 미반영

근본 원인: 이전 개발 과정의 찌꺼기가 산재하고, 라이브 코드가 백테스트와 독립적으로 진화하여 두 시스템이 silently 불일치.

## 대안

1. 발견되는 gap만 부분 수정 → 끝없는 패치, 또 다른 silent gap 잠복
2. 1주차 모니터링 계속 → baseline 없는 비교, 의미 없음
3. 백테스트 기준 재조립 (선택) → clean start, 모든 부속 의도적 추가

## 결정 이유
부분 수정으로는 "다른 영역에 또 뭐가 있을까" 불안이 해소 안 됨. 백테스트가 유일한 검증된 baseline이므로 거기서 시작하는 게 정공법.

## 상태
ACCEPTED
