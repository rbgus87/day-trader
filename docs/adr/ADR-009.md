# ADR-009: 비용 모델 통일 + 거래세 0.18 → 0.15 보정

## 일자
2026-04-15

## 결정
1. **거래세 0.18% → 0.15%로 정정** — 2024 정부 정책 반영
2. **공유 cost_model 모듈 신설** (`core/cost_model.py`) — backtester와 PaperOrderManager가 동일 로직 사용
3. **PaperOrderManager에 비용 모델 도입** — slipped_price + 수수료/세금 반영하여 `trades.pnl` 이 비용 차감 후 값
4. **슬리피지 0.03%는 현행 유지** — 추정값. 페이퍼 1주차 회고 후 실측 대비 별도 ADR

## 배경
Phase 3-A 조사 (`docs/spec/live_baseline_comparison.md` §I)에서 발견:
- backtester는 `commission/slippage/tax` 완전 반영
- PaperOrderManager는 `price × qty` 그대로 기록 (비용 0건)
- 거래세 0.18%는 구 정책 기준 (실제 0.15%)

이로 인해 baseline ↔ paper PnL 직접 비교 불가 + 거래세 과대 측정.

## 대안
| 안 | 장단점 |
|---|---|
| A. 정정 0.18→0.15만 | 세금 보정. 그러나 paper는 여전히 비용 0 |
| B. 비용 모델만 통일 (tax 그대로) | paper 정합 확보. 그러나 0.18% 과대 측정 |
| C. 둘 다 유지 (현행) | silent gap |
| **D. 둘 다 정정 (선택)** | **세금 정확 + paper 정합 — baseline 직접 비교 가능** |

## 구현 (5개 커밋)

### 1) tax 0.18 → 0.15 (`5d6c6e0`)
- `config.yaml backtest.tax`: 0.0018 → 0.0015
- `config/settings.py BacktestConfig.tax`: 동일
- `tests/test_settings.py`: assertion 동기화

**새 baseline 측정 (2025-04-01 ~ 2026-04-10, 60종목)**:
| 지표 | 이전 (tax 0.18%) | **신규 (tax 0.15%)** |
|---|---|---|
| 거래수 | 185 | 185 |
| PF | 2.91 | **2.99** |
| 총 PnL | +288,812 | **+294,842** |
| 거래당 PnL | +1,561 | **+1,594** |
| 청산 분포 | 168/14/3/0 | 168/14/3/0 (동일) |
| PF>1 종목 | 33/60 | 33/60 (동일) |

세금 절감 기여: 185회 매도 × 0.03% × 평균 거래금액 ≈ +6,030원 (실측 부합).

### 2) cost_model.py 신설 + backtester 리팩토링 (`3c06e6f`)
- `core/cost_model.py`:
  - `TradeCosts` dataclass (commission/slippage/tax rate)
  - `apply_buy_costs(price, costs) → (slipped, net_entry)`
  - `apply_sell_costs(price, costs) → (slipped, net_exit)`
  - `from_backtest_config()` 팩토리
- `backtest/backtester.py`:
  - `self._costs = TradeCosts.from_backtest_config(...)`
  - 6곳 인라인 비용 계산 → helper 호출로 교체
  - **로직 변경 0 — PF 2.99 / PnL +294,842 완전 재현 (1원 차이 없음)**

### 3) PaperOrderManager 비용 모델 도입 (`bc6ef44`)
- `__init__`에 `backtest_config` 파라미터
- `self._net_entry_map: dict[ticker, float]` — 주당 net_entry 저장
- `execute_buy`: slipped_price로 DB 기록 + `_net_entry_map` 갱신
- `_simulate_order` (sell): apply_sell_costs로 net_exit 계산, pnl = (net_exit - net_entry) × qty
- DB `trades.pnl`이 **비용 차감 후 실제 값**
- `engine_worker`: PaperOrderManager 인스턴스화에 `backtest_config` 전달

### 4) cost_model 단위 테스트 (`a2c496f`)
- 9 tests / all pass
- 매수/매도/왕복 시나리오 커버
- 왕복 손실률 ≈ -0.24% 검증 (commission×2 + tax + slippage×2)

### 5) ADR-009 + CLAUDE.md (이번 커밋)

## 영향

### baseline 비교 가능해짐
이전: backtester PF 2.91 vs paper DB의 raw PnL — 비교 불가
현재: backtester PF 2.99 vs paper DB pnl (비용 차감) — **직접 비교 가능**

### 4/13~4/15 기존 trades 데이터
- 비용 미반영 상태로 보존 (소급 보정 **안 함**)
- 이전 데이터는 페이퍼 1주차 회고에 포함 안 함 — 단순 동작 검증용

### order_manager.py (실매매)
- 이번 작업 범위 외 (사용자 결정)
- 실매매는 키움 API 응답값이 이미 슬리피지 반영. 수수료/세금 정산은 별도 로직 필요 (후속 ADR 가능)

## 미해결 / Follow-up
- **슬리피지 0.03% 실측 검증** — 페이퍼 1주차 체결 데이터로 평균 슬리피지 측정 후 ADR 가능
- **실매매 비용 반영** — 키움 계좌별 수수료율 다를 수 있어 별도 계산 (ADR 후보)
