### 근본 원인

- 시스템 거래 빈도: 연 279건 / 41종목 = **종목당 연 6.8건**
- 월 0.57건/종목은 "엄선된 진입만 하는" 시스템의 특성
- Walk-Forward 검증 기간에 충분한 거래가 쌓이려면 **최소 10개월 이상** 필요
- 현재 데이터 12개월 중 10개월을 검증에 쓰면 학습 2개월 → 의미 없음

## 대안 평가

### 대안 1: 7+5 분할 (37종목) — 최선의 타협

- 학습: 2025-04 ~ 2025-10 (143일, 37종목)
- 검증: 2025-11 ~ 2026-04 (111일, 37종목)
- 추정 거래: 학습 ~149건, 검증 ~105건
- **검증 100건 수준 — 방향성 확인은 가능, 통계적 확신은 제한**

실행 방법:
1. 학습 기간으로 backtest_single.py 실행 → PF 측정
2. 검증 기간으로 실행 → PF 측정
3. 학습 PF vs 검증 PF 비교 (큰 차이 없으면 과적합 아님)

### 대안 2: Rolling Walk-Forward (3개월 학습 + 1개월 검증 × 9회)

- 학습: 직전 3개월, 검증: 다음 1개월
- 총 9개 윈도우 (2025-07 ~ 2026-04)
- 각 윈도우 거래수: ~23건 (매우 적음)
- 전체 합산: ~210건 → **통계 OK**
- 단, 각 윈도우별 PF는 의미 없음 (23건으로 PF 산출)

### 대안 3: Walk-Forward 스킵 → 페이퍼 직행

근거:
- 현재 시스템 파라미터가 광범위한 그리드 서치로 검증됨
- 각 파라미터의 "주변 값 안정성" 확인됨 (sweet spot 좌우 모두 테스트)
- 12개월 데이터 전체로 PF 3.28 측정
- 과적합 징후: volume_ratio/buy_time_end의 sharp sweet spot은 위험 신호이나, 이 값들은 시스템의 근본적 엣지(진짜 돌파 필터)이지 데이터에 피팅된 것이 아님
- 페이퍼 1~2주가 실질적인 out-of-sample 검증

## 권장

**대안 1 (7+5 분할) 실행 + 검증 PF 확인 후 페이퍼 진행**

이유:
1. 실행 비용 낮음 (backtest_single 2회 실행)
2. 100건이라도 PF 방향성 확인 가능 (3.0 vs 1.0 정도의 차이는 감지)
3. 학습 PF ≈ 검증 PF이면 과적합 가능성 낮음
4. 검증 PF가 크게 하락(PF < 1.5)하면 과적합 경고
5. 어차피 페이퍼가 진짜 out-of-sample — Walk-Forward는 사전 점검

### 판정 기준

| 검증 PF | 판정 | 다음 단계 |
|---------|------|---------|
| >= 2.0 | 과적합 아님 | 페이퍼 진행 |
| 1.5 ~ 2.0 | 경계 | 페이퍼 진행하되 주의 |
| 1.0 ~ 1.5 | 약한 과적합 | 원인 분석 후 결정 |
| < 1.0 | 과적합 의심 | 파라미터 재검토 |
